Priority Guide

P1 — Critical (96 tests) Core functionality or security. Must pass before any release. Failures block deployment.
P2 — Important (144 tests) Key features covering most use-cases. Must pass before production release.
P3 — Nice to Have (15 tests) Edge cases and cosmetic details. Target for post-launch polish.
